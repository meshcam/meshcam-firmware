# Contributing

MeshCam is developed on a private tree and released here in batches — this
GitHub repo is the canonical public mirror, and releases land as squashed
commits. This is deliberate: it keeps the public history readable and keeps
review bandwidth for humans (unsolicited bulk or AI-generated PRs will be
closed unread).

How to actually contribute:

- **Bugs, questions, field reports** → Issues / Discussions. Actively
  watched; this is the front door.
- **Small obvious fixes** (docs, typos, clear one-liners) → a PR is fine; it
  may be applied to the private tree and credited in release notes rather
  than merged directly.
- **Anything bigger** → open an issue FIRST so we agree on direction before
  you invest the time.
- **CLA**: code contributions require a lightweight Contributor License
  Agreement on your first PR (you keep your copyright; the grant is what
  lets the project offer commercial licenses that fund development).
